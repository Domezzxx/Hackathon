# 🦷 DentBot AI — คลินิกทันตกรรม มทส.

แอป React Native (Expo) ผู้ช่วยทันตกรรม: ปรึกษาอาการกับ AI, วิเคราะห์รูปฟัน, จองคิว, จัดการนัดหมาย และ **คุยกับทันตแพทย์ตัวจริงแบบเรียลไทม์**

## ฟีเจอร์

| แท็บ | ทำอะไร |
|------|--------|
| **สนทนา AI** | แชตกับ DentBot (Google Gemini 2.5 Flash) ปรึกษาอาการ + ส่งรูปฟันให้ AI วิเคราะห์ |
| **คุยกับหมอ** 🆕 | เลือกหมอจาก**ทีมทันตแพทย์** (เชี่ยวชาญต่างกัน, ออนไลน์/ออฟไลน์) → แชตตัวจริง ส่งข้อความ/รูปฟัน เห็น "หมอกำลังพิมพ์" |
| **จองคิว** | เลือกบริการ/วัน/เวลา จองให้ตัวเองหรือคนอื่น |
| **Admin** | จัดการนัดหมาย (ยืนยัน/ยกเลิก) + **แชตคนไข้** (หมอตอบจากที่นี่) |

## การรัน

```bash
npm install            # ติดตั้ง dependencies (ทำแล้ว)
npm start              # เปิด Expo → สแกน QR ด้วยแอป Expo Go บนมือถือ
# หรือ
npm run web            # เปิดบนเบราว์เซอร์
npm run android        # เปิดบน Android emulator
```

## API Key

ตั้งค่าใน `.env` (อยู่ใน `.gitignore` แล้ว — ไม่ commit):

```
EXPO_PUBLIC_GEMINI_API_KEY=your_key_here
```

ขอ key ฟรีที่ https://aistudio.google.com/apikey · ดูตัวอย่างใน `.env.example`

## เดโม "คุยกับหมอ" บนเครื่องเดียว

1. แท็บ **คุยกับหมอ** → เลือกหมอจากรายชื่อ (ออนไลน์/ออฟไลน์) → พิมพ์อาการ/ส่งรูปฟัน → กดส่ง
2. กดปุ่ม **Admin** (มุมขวาบน, มี badge แจ้งแชตใหม่) → แท็บ **แชตคนไข้**
3. เลือกหมอจาก selector ด้านบน (badge บอกว่าหมอคนไหนมีแชตใหม่) → พิมพ์ตอบในนามหมอคนนั้น → กดส่ง
4. ปิด Admin กลับมาแท็บ **คุยกับหมอ** → เห็นคำตอบหมอเด้งเข้ามา (badge แจ้งเตือนบนแท็บ)

> หมอแต่ละคนมี thread แยกกัน · Admin toggle ออนไลน์/ออฟไลน์ รายคนได้ (จำลองหมอที่ Active / ไม่ Active)

> ใน AI chat กดปุ่ม **"คุยกับหมอ"** เพื่อส่งต่อจากบอทไปหาหมอจริงได้ทุกเมื่อ

## โครงสร้าง

```
App.js                 ศูนย์กลาง state + แท็บ + ปุ่ม Admin
ChatScreen.js          แชต AI (Gemini) + ส่งต่อหาหมอ
DoctorTab.js           รายชื่อทีมหมอ (ฝั่งคนไข้) → เข้าแชตรายคน  🆕
DoctorChatScreen.js    แชตคุยกับหมอ 1 คน (ฝั่งคนไข้)  🆕
QueueScreen.js         จองคิว
AdminScreen.js         จัดการนัดหมาย + แชตคนไข้ (เลือกหมอ → ตอบ)  🆕
constants.js           บริการ/ราคา/เวลา + ทีมหมอ DOCTORS + API key
assets/                รูปบริการ + ไอคอน
```

## หมายเหตุ

ระบบแชตหมอใช้ shared state ใน `App.js` (เดโมเครื่องเดียว). ต่อยอดเป็น multi-device จริงได้ด้วย Firebase Realtime DB หรือ WebSocket โดยเปลี่ยนแค่ `handlePatientSendToDoctor` / `handleDoctorReply` ให้ยิงผ่าน backend
